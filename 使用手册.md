## 环境要求

| 依赖 | 版本/说明 |
|---|---|
| Python | 3.11.15 |
| Conda 环境 | `separation` |
| CUDA | 12.9 (运行时) / 12.8 (编译) |
| GPU | NVIDIA RTX 5070 Laptop (8GB VRAM) |
| PyTorch | 2.11.0+cu128 (GPU 版) |
| FFmpeg | 已安装 (用于解码 MP3/FLAC 等格式) |

> **注意**：必须使用 GPU 版 PyTorch。CPU 版本虽然能跑，但速度会慢 10-50 倍。

## 安装

### 1. 激活 Conda 环境

```bash
conda activate separation
```

### 2. 安装 GPU 版 PyTorch

> **关键步骤**：不要直接用 `pip install demucs`，它会自动装 CPU 版 PyTorch。

```bash
# 先装 GPU 版 PyTorch (CUDA 12.8 编译，兼容 CUDA 12.9 运行时)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128
```

### 3. 安装 Demucs 及其他依赖

```bash
# 安装 Demucs (此时 PyTorch 已是 GPU 版，不会被覆盖)
pip install demucs

# 安装 soundfile (用于保存 WAV，替代有 DLL 兼容问题的 torchcodec)
pip install soundfile
```

### 4. 验证 GPU 是否就绪

```bash
python -c "import torch; print('CUDA available:', torch.cuda.is_available()); print('GPU:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'N/A')"
```

应该输出：

```
CUDA available: True
GPU: NVIDIA GeForce RTX 5070 Laptop GPU
```

### 5. 克隆项目

```bash
git clone https://github.com/WuChangqing1/SeparationPersonFromBeat.git
cd SeparationPersonFromBeat
```

## 快速开始

```bash
conda activate separation
cd d:\CodingData\Python\Tools\Separation

# 人声+伴奏分离 (最常用)
python -m src.main -i 你的歌曲.mp3 -o ./output --two-stems vocals

# 四音源全部分离
python -m src.main -i 你的歌曲.mp3 -o ./output
```

> **首次运行**会自动下载模型文件（约 80MB），存储在 `C:\Users\<用户名>\.cache\torch\hub\checkpoints\`，仅需下载一次。

## 命令行参数

```
usage: separation [-h] -i INPUT [-o OUTPUT] [-m MODEL] [--device DEVICE]
                  [--shifts SHIFTS] [--overlap OVERLAP] [--two-stems STEM]
                  [--segment SEGMENT]
```

### 必选参数

| 参数 | 说明 |
|---|---|
| `-i`, `--input` | 输入音频文件路径。支持 WAV、MP3、FLAC 等常见格式 |

### 可选参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `-o`, `--output` | `./output` | 输出目录。会在其下创建 `{模型名}/{文件名}/` 子目录 |
| `-m`, `--model` | `htdemucs` | 模型名称。可选: `htdemucs`, `htdemucs_ft`, `htdemucs_6s` |
| `--device` | 自动检测 | 设备选择: `cuda` (GPU) 或 `cpu`。默认有 GPU 就用 GPU |
| `--shifts` | `1` | 等变稳定化随机偏移次数。越大质量越好，但耗时线性增长。论文使用 `10` |
| `--overlap` | `0.25` | 分段重叠比例 (0~1)。提高可减少分段处的伪影 |
| `--two-stems` | 无 | 指定一个音轨名 (如 `vocals`)，只输出该轨 + 其余混合轨。详见下表 |
| `--segment` | 自动 | 每处理分段的帧数。**显存不足时调小此值**（如 `6` 或 `4`） |

### `--two-stems` 支持的音轨名

| 值 | 输出 1 | 输出 2 |
|---|---|---|
| `vocals` | `vocals.wav` (人声) | `no_vocals.wav` (伴奏 = drums+bass+other) |
| `drums` | `drums.wav` (鼓组) | `no_drums.wav` (除鼓外所有) |
| `bass` | `bass.wav` (贝斯) | `no_bass.wav` (除贝斯外所有) |
| `other` | `other.wav` (其他) | `no_other.wav` (除其他外所有) |

## 使用示例

### 示例 1：基础四音源分离

```bash
python -m src.main -i music.mp3 -o ./output
```

输出：
```
output/htdemucs/music/
├── vocals.wav    # 人声
├── drums.wav     # 鼓组
├── bass.wav      # 贝斯
└── other.wav     # 其他
```

### 示例 2：仅分离人声和伴奏 (两轨)

```bash
python -m src.main -i music.mp3 -o ./output --two-stems vocals
```

输出：
```
output/htdemucs/music/
├── vocals.wav    # 人声
└── no_vocals.wav # 伴奏
```

### 示例 3：高质量分离

```bash
# shifts=5 会产生 5 次随机偏移，取平均，质量更高但慢 5 倍
python -m src.main -i music.wav --shifts 5
```

### 示例 4：显存不足时

```bash
# 如果 GPU 显存不足 (8GB 以下处理长音频)，减小 segment
python -m src.main -i long_music.mp3 --segment 6
```

### 示例 5：CPU 模式

```bash
# 没有 GPU 或 GPU 不兼容时
python -m src.main -i music.mp3 --device cpu
```

> **注意**：CPU 模式非常慢，一首 5 分钟的歌可能需要 5-15 分钟。

### 示例 6：使用其他模型

```bash
# 微调版 HTDemucs (某些场景质量更好)
python -m src.main -i music.mp3 -m htdemucs_ft

# 6 音源版 (多出 guitar 和 piano)
python -m src.main -i music.mp3 -m htdemucs_6s
```

### 示例 7：处理 FLAC 无损音频

```bash
python -m src.main -i lossless.flac -o ./output --two-stems vocals
```

### 示例 8：指定输出位置

```bash
python -m src.main -i music.mp3 -o D:/MyMusic/Separated
```

## 输出文件说明

### 音轨含义

| 音轨 | 英文 | 包含内容 |
|---|---|---|
| `vocals` | Vocals | 主唱人声、和声 |
| `drums` | Drums | 底鼓、军鼓、镲片、通鼓等打击乐器 |
| `bass` | Bass | 贝斯吉他、合成器低音、808 等低频乐器 |
| `other` | Other | 吉他、钢琴、弦乐、合成器、管乐等所有剩余乐器 |

### 文件格式

- **格式**：WAV (PCM 16-bit)
- **采样率**：44100 Hz
- **声道**：立体声 (2 channels)
- **位深**：16-bit

## 模型说明

| 模型 | 音源数 | 说明 |
|---|---|---|
| `htdemucs` (默认) | 4 | Hybrid Transformer Demucs，速度与质量的最佳平衡 |
| `htdemucs_ft` | 4 | 微调版，在某些音乐类型上质量更好 |
| `htdemucs_6s` | 6 | 6 音源版: vocals, drums, bass, other, guitar, piano |

### 模型文件缓存

模型首次使用时会从 Meta 服务器下载，存储到：

```
C:\Users\<用户名>\.cache\torch\hub\checkpoints\
```

之后每次运行直接加载缓存，无需重复下载。

## 性能调优

### 速度 vs 质量

| 参数组合 | 速度 | 质量 | 适用场景 |
|---|---|---|---|
| `--shifts 1` (默认) | 最快 | 良好 | 日常使用、快速预览 |
| `--shifts 5` | 慢 5 倍 | 更好 | 正式输出 |
| `--shifts 10` | 慢 10 倍 | 最佳 | 论文级质量 |

### 显存优化

| 参数 | 效果 |
|---|---|
| `--segment 4` | 将音频切成约 4 秒片段处理，显存需求大幅降低 |
| `--segment 6` | 约 6 秒片段，平衡选择 |
| `--device cpu` | 不用 GPU 显存，但极慢 |

### 实测性能 (RTX 5070 Laptop 8GB)

| 音频长度 | `--shifts 1` | `--shifts 5` | `--shifts 10` |
|---|---|---|---|
| 3 分钟 | ~5 秒 | ~25 秒 | ~50 秒 |
| 5 分钟 | ~8 秒 | ~40 秒 | ~80 秒 |
| 10 分钟 | ~16 秒 | ~80 秒 | ~160 秒 |

## 常见问题

### Q1: 提示 `No module named 'torchcodec'`

**原因**：torchaudio 2.11.0 的 `save()` 依赖 torchcodec，但 Windows 上有 DLL 兼容问题。

**解决**：本项目已改用 `soundfile` 保存音频。确保安装了 soundfile：

```bash
pip install soundfile
```

### Q2: `CUDA available: False`

**原因**：装了 CPU 版 PyTorch。

**解决**：
```bash
# 卸载 CPU 版
pip uninstall torch torchvision torchaudio -y
# 重新装 GPU 版
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128
```

### Q3: 显存不足 (CUDA out of memory)

**解决**：调小 `--segment` 参数：
```bash
python -m src.main -i music.mp3 --segment 4
```

### Q4: 输出音频有杂音或失真

**原因**：可能是音频本身质量问题或 MP3 编码伪影。

**尝试**：
1. 使用 WAV/FLAC 无损格式输入
2. 增加 `--shifts` 值 (如 5)
3. 尝试 `-m htdemucs_ft` 微调版模型

### Q5: FFmpeg 报错 / 无法读取文件

**检查** FFmpeg 是否在系统 PATH 中：
```bash
ffmpeg -version
```

如果没有，从 https://ffmpeg.org/download.html 下载并添加到 PATH。

### Q6: 首次运行下载模型失败

**原因**：网络无法访问 Meta 的 CDN。

**解决**：手动下载模型文件放到缓存目录：
1. 浏览器打开: https://dl.fbaipublicfiles.com/demucs/hybrid_transformer/955717e8-8726e21a.th
2. 下载后放到 `C:\Users\<用户名>\.cache\torch\hub\checkpoints\`
